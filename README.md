# 🚀 Сервис управления рассылками

**Django-based email marketing automation system with role-based access control and comprehensive analytics**

## 📋 О проекте

Профессиональный сервис для создания, управления и анализа email-рассылок. Разработан с нуля с использованием Django, реализует многоуровневую систему доступа, автоматическую обработку статусов и детальную статистику эффективности кампаний.

### 🎯 Ключевые возможности

#### 👥 Управление пользователями и ролями
- **Трехуровневая система доступа:** Пользователь → Менеджер → Администратор
- Подтверждение email при регистрации (24-часовая валидность токена)
- Блокировка/разблокировка пользователей менеджерами
- Гибкое управление ролями и правами

#### 📧 Управление рассылками
- **CRUD операции** для клиентов, сообщений и рассылок
- **Автоматическое управление статусами** (Создана → Запущена → Завершена)
- Фильтрация по времени начала/окончания
- Поддержка массовых и целевых рассылок
- Возможность ручного запуска из интерфейса

#### 📊 Статистика и аналитика
- **Процент успешных/неуспешных отправок** с визуализацией (Bootstrap прогресс-бары)
- Детальный лог каждой попытки с ответом сервера
- Счетчики успешных/неудачных отправок для каждой рассылки
- Общая статистика по всем рассылкам пользователя
- Отображение последних 5 рассылок на главной странице

#### 🔒 Безопасность
- **Кастомная модель пользователя** (email как логин)
- Защита от несанкционированного доступа (декораторы `@login_required`, `@manager_required`)
- Валидация владельца объектов (пользователь видит только свои данные)
- Контекст-процессор для автоматического обновления статусов рассылок
- Блокировка пользователей без удаления данных

## 🛠 Технический стек

| Компонент | Технологии |
|-----------|------------|
| **Backend** | Django 5.2.7, Python 3.13 |
| **Database** | SQLite (dev) / PostgreSQL (prod) |
| **Frontend** | Bootstrap 5, HTML/CSS, Django Templates |
| **Email** | Django SMTP, console backend для dev |
| **Auth** | Django Auth + кастомные профили |
| **Caching** | LocMemCache (900 сек) |
| **Deployment** | Poetry, python-dotenv |

## 🚀 Быстрый старт

### 1️⃣ Клонирование и настройка

```
git clone https://github.com/morozoVik/mailing_service_project.git
cd mailing_service_project
```

### 2️⃣ Установка зависимостей

```
# Poetry (рекомендуется)
poetry install

# или pip
pip install -r requirements.txt
```

### 3️⃣ Настройка окружения

```
cp .env.example .env
# Отредактируйте .env, добавьте секретный ключ
```

### 4️⃣ База данных и миграции

```
python manage.py makemigrations
python manage.py migrate
```

### 5️⃣ Создание суперпользователя

```
python manage.py createsuperuser
```

### 6️⃣ Запуск сервера

```
python manage.py runserver
```

## 🔧 Управление данными

### Создание тестовых данных

```
python manage.py create_test_data
# Создает пользователя testuser / testpass123
# + 3 клиента, 1 сообщение, 1 рассылку
```

### Создание менеджера

```
python manage.py create_manager
# manager / manager123
```

### Настройка прав доступа

```
python manage.py create_managers_group
# Создает группу "Менеджеры" со всеми правами на рассылки
```

### Просмотр всех данных

```
python manage.py debug_data
```

## 📊 API и функциональность

### Основные маршруты

| URL	| Метод	| Описание |
|-------|-------|----------|
|```/```|	GET	| Главная страница со статистикой
```/clients/```|	GET/POST	|Список/создание клиентов
```/mailings/```|	GET/POST	|Список/создание рассылок
```/mailings/<id>/send/```|	POST	|Ручной запуск рассылки
```/users/```|	GET	|Список пользователей (только менеджер)
```/users/<id>/toggle-block/```|	POST	|Блокировка пользователя

### Модели данных

**Client** - получатель рассылки
- email, full_name, comment, owner

**Message** - шаблон письма
- subject, body, owner

**Mailing** - настройки рассылки
- start_time, end_time, status, message, clients, owner

**MailingAttempt** - логи отправки
- attempt_time, status, server_response, mailing

- Создание/редактирование/удаление СВОИХ клиентов

## 📈 Особенности реализации
Автоматическое обновление статусов
Контекст-процессор update_mailing_statuses автоматически обновляет статусы рассылок при каждом запросе пользователя, используя метод update_status().

### Отправка email

- Использует Django send_mail с обработкой исключений
- Каждая попытка логируется в MailingAttempt
- Поддержка консольного бэкенда для разработки

### Статистика в реальном времени

- Процент успеха/неудач
- Количество активных/завершенных/созданных рассылок
- Уникальные клиенты
- Детализация по каждой рассылке
